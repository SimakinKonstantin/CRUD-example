### Примечания
Используемая БД: PostgreSQL.
Для работы с миграциями использовался goose: https://github.com/pressly/goose

### Шаги для запуска:
1. Определить переменную окружения ```POSTGRES_CONNECT_URL``` со ссылкой на подключение к БД.
2. В ```./internal/config/config.go``` указать значение ServerAddr - адреса сервера.
3. Запустить миграцию, создающую таблицу через ```cd ./migrations && goose up``` либо
вручную выполнив sql код до ```-- +goose Down```.
4. Запустить ```./cmd/main.go```.

### Особенности:
1. В задании было сказано, что должна быть функция вывода списка сотрудников для указанного отдела.
Но не было сказано, как именно. В реализованной версии по имени отдела (имя отдела - Primary Key).
2. Не было сказано, как должны изменяться Departments при изменении сотрудника. Было решено, что при отправке нового 
значения Department при изменении пользователя существующий Department будет изменен.
3. Был задан уточняющий вопрос по заданию: "Все методы должны быть реализованы в виде HTTP запросов в 
формате JSON", на что был получен ответ: "Можешь использовать верную спецификацию и тогда для GET использовать 
не JSON а Query параметры". Поэтому используется не только чистые JSON запросы для указания параметров,
но также и Query параметры.

### Задание:
Web-Сервис сотрудников, сделанный на Golang
Сервис должен уметь:
1. Добавлять сотрудников, в ответ должен приходить Id добавленного
   сотрудника. 
2. Удалять сотрудников по Id.
3. Выводить список сотрудников для указанной компании. Все доступные поля. 
4. Выводить список сотрудников для указанного отдела компании. Все доступные
   поля.
5. Изменять сотрудника по его Id. Изменения должно быть только тех
   полей, которые указаны в запросе.
   
Модель сотрудника:
   ```
   {
       Id int
       Name string
       Surname string
       Phone string
       CompanyId int
       Passport {
           Type string
           Number string
       }
       Department {
           Name string
           Phone string
       }
   }
   ```
   
   Все методы должны быть реализованы в виде HTTP запросов в формате JSON. БД: любая
